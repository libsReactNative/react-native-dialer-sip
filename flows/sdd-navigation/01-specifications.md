## Overview

The Navigation module provides stack-based navigation using react-native-deprecated-custom-components Navigator with Redux state management for route tracking and screen transitions.

## Functional Requirements

### FR-1: Store Navigator Reference

**Description**: Store reference to Navigator component for imperative navigation.

**Action**: `ref(nav)`

**Process**:
1. Dispatch REF action with Navigator reference
2. Store in state.navigation.ref

**Usage**:
```javascript
componentDidMount() {
  this.props.onNavigatorMount(this.navigatorView)
}

// In actions
function actions(dispatch) {
  return {
    onNavigatorMount: (ref) => {
      dispatch(Navigation.ref(ref))
    }
  }
}
```

### FR-2: Navigate To Route

**Description**: Push new route onto navigation stack.

**Action**: `goTo(route)`

**Inputs**:
- `route`: Object with `name` property and optional additional data

**Process**:
1. Get Navigator ref from state
2. Call `ref.push(route)` (imperative)
3. Dispatch NAVIGATE_TO action

**State Transformation**:
```javascript
current: route
previous: current
history: history.concat(current)
drawer: false
```

### FR-3: Navigate Back

**Description**: Pop current route from stack and return to previous.

**Action**: `goBack()`

**Process**:
1. Get Navigator ref from state
2. Call `ref.pop()` (imperative)
3. Wait 500ms (animation delay)
4. Dispatch NAVIGATE_BACK action

**State Transformation**:
```javascript
current: previous
previous: history[history.length - 1]
history: history.pop()
drawer: false
```

**Delay**: 500ms (sync with animation)

### FR-4: Navigate And Replace

**Description**: Replace current route with new route (with tab logic).

**Action**: `goAndReplace(route)`

**Inputs**:
- `route`: New route object

**Tab Logic**:
```javascript
const same = ['conversations', 'dialer', 'history', 'settings'].indexOf(route.name) !== -1 &&
  ['conversations', 'dialer', 'history', 'settings'].indexOf(tab) !== -1

if (ref && !same) {
  ref.replace(route)
}
```

**Process**:
1. Get current tab name
2. If both current and new are tab routes AND same: don't replace
3. Otherwise: call `ref.replace(route)`
4. Dispatch NAVIGATE_REPLACE action

**State Transformation**:
```javascript
current: route
history: [...history, route] (replaces last)
drawer: false
```

### FR-5: Drawer Navigation

**Description**: Open/close drawer navigation.

**Actions**: `openDrawer()`, `closeDrawer()`

**Process**:
- Open: Dispatch OPEN_DRAWER action
- Close: Dispatch CLOSE_DRAWER action

**State Transformation**:
```javascript
// Open
drawer: "VISIBLE"

// Close
drawer: false
```

### Render Scene

```javascript
renderScene(route, navigator) {
  switch (route.name) {
    case 'launch':
      return (<LaunchScreen />)
    case 'conversations':
    case 'contacts':
    case 'history':
    case 'settings':
    case 'dialer':
      return (<Viewport navigator={navigator}/>)
    case 'call':
      return (<CallScreen />)
    case 'account':
      return (<AccountScreen />)
    case 'network_settings':
      return (<NetworkSettingsScreen />)
    case 'media_settings':
      return (<MediaSettingsScreen />)
    default:
      return (<View />)
  }
}
```

## Usage Patterns

### Initial Navigation

```javascript
// In app/index.js
store.dispatch(async (dispatch, getState) => {
  await dispatch(App.init())
  
  // Determine initial route
  let route = {name: 'settings'}
  const {calls, accounts} = getState().pjsip

  for (const id in accounts) {
    route = {name: 'dialer'}
    break
  }

  for (const id in calls) {
    const call = calls[id]
    if (call.getState() === "PJSIP_INV_STATE_INCOMING") {
      route = {name: 'call', call}
      break
    }
  }

  dispatch(Navigation.goAndReplace(route))
})
```

### Navigation After Call End

```javascript
// In CallScreen actions
onCallEnd(call) {
  setTimeout(() => {
    dispatch(async function (dispatch, getState) {
      const calls = getState().pjsip.calls
      const route = getState().navigation.current
      
      const doDirectRoute = () => dispatch(Navigation.goBack())
      const doRoute = (call) => {
        if (calls.hasOwnProperty(call.getId())) return
        
        for (const id in calls) {
          if (calls.hasOwnProperty(id)) {
            return dispatch(Navigation.goAndReplace({name: 'call', call: calls[id]}))
          }
        }
        
        dispatch(Navigation.goBack())
      }

      if (route.name !== 'call') return
      
      if (route.call instanceof Promise) {
        route.call.then(doRoute, doDirectRoute)
      } else {
        doRoute(route.call)
      }
    })
  }, 3000)
}
```

### Making a Call (Navigation)

```javascript
// In pjsip.js makeCall()
dispatch(Navigation.goTo({name: 'call', call}))
```

### Incoming Call (Navigation)

```javascript
// In pjsip.js onCallReceived()
const state = getState()
if (state.navigation.current.name != 'call' && state.pjsip.appState == 'active') {
  dispatch(Navigation.goTo({name: 'call', call}))
}
```

## Technical Debt & Risks

### 🔴 Deprecated Library

**Issue**: Uses `react-native-deprecated-custom-components` from GitHub

**Risk**: Library not maintained, may break with React Native updates

**Evidence**:
```javascript
import {Navigator} from '../../app/old/navigator/CustomComponents'
// import {Navigator} from 'react-native-deprecated-custom-components'
```

**Commented Alternatives**:
```javascript
// import {createAppContainer} from 'react-navigation'
// import {createStackNavigator} from 'react-navigation-stack'
// const AppNavigator = createStackNavigator({...})
```

**Recommendation**: Migrate to react-navigation v6+

### 🟡 500ms Delay in goBack

**Issue**: Artificial delay in goBack action

**Code**:
```javascript
setTimeout(function () {
  dispatch({type: NAVIGATE_BACK})
}, 500)
```

**Impact**: Navigation feels sluggish
**Recommendation**: Remove delay or make configurable

### 🟡 Tab Route Logic

**Issue**: Hardcoded tab route names in goAndReplace

**Code**:
```javascript
const same = ['conversations', 'dialer', 'history', 'settings'].indexOf(route.name) !== -1
```

**Risk**: Must update when adding new tab routes
**Recommendation**: Centralize tab route configuration
