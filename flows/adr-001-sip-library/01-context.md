## Context

The application requires a SIP (Session Initiation Protocol) library for Voice over IP (VoIP) communication in React Native. The library must:

- Support iOS and Android platforms
- Handle SIP account registration
- Manage call lifecycle (make, answer, hold, transfer, hangup)
- Support push notifications for incoming calls
- Integrate with native call kits (CallKit on iOS)

## Decision Drivers

- **Platform Support**: Must work on both iOS and Android
- **PJSIP Protocol**: Full SIP protocol implementation via PJSIP
- **Push Notifications**: Critical for incoming calls when app is backgrounded
- **CallKit Integration**: Native call UI on iOS for better UX
- **Maintenance**: Active maintenance and bug fixes
- **React Native Compatibility**: Works with React Native 0.61.5

## Considered Options

### Option 1: react-native-pjsip

**Description**: Original PJSIP wrapper for React Native.

**Pros**:
- Well-established library
- Good documentation
- Active community (at the time)

**Cons**:
- May have unresolved issues or slower maintenance
- Potential compatibility issues with newer React Native versions

### Option 2: react-native-sip2 (CHOSEN)

**Description**: Fork/continuation of react-native-pjsip with improvements.

**Pros**:
- More recent maintenance
- Bug fixes from original library
- Better compatibility with newer React Native versions
- Same API as react-native-pjsip (drop-in replacement)

**Cons**:
- Smaller community than original
- Less documentation specific to this fork

### Option 3: react-native-callkit only

**Description**: Use CallKit directly without PJSIP.

**Pros**:
- Native iOS integration
- No third-party SIP library dependency

**Cons**:
- iOS only (no Android support)
- Requires separate SIP implementation
- More development effort
- **REJECTED**: Does not meet cross-platform requirement

### Option 4: Custom native module

**Description**: Build custom native modules wrapping PJSIP.

**Pros**:
- Full control over implementation
- Can optimize for specific use cases

**Cons**:
- Significant development effort
- Requires native iOS and Android expertise
- Ongoing maintenance burden
- **REJECTED**: Time and cost prohibitive

## Decision

**Selected**: **Option 2 - react-native-sip2**

**Rationale**:

1. **Drop-in Replacement**: Uses same API as react-native-pjsip, minimizing migration effort
2. **Active Maintenance**: More recent updates and bug fixes
3. **Cross-Platform**: Supports both iOS and Android
4. **PJSIP Foundation**: Leverages mature PJSIP protocol implementation
5. **Push Notification Support**: Built-in support for VoIP push notifications
6. **Package Availability**: Available via npm (`react-native-sip2@^3.0.2`)

**Evidence from Code**:


## Consequences

### Positive

1. **Functionality**: Full SIP protocol support achieved
2. **Cross-Platform**: Single codebase for iOS and Android
3. **Push Notifications**: VoIP push notifications work on iOS
4. **Familiar API**: Team familiar with react-native-pjsip API can adapt quickly
5. **NPM Distribution**: Easy installation and versioning

### Negative

1. **CallKit Limitation**: CallKit doesn't support multiple simultaneous calls
   - **Workaround**: Code handles single active call constraint
   - **Evidence**: Commented out CallKit integration in pjsip.js

2. **Smaller Community**: Less community support than original library
   - **Mitigation**: Can fall back to react-native-pjsip documentation

3. **Dependency Risk**: Reliance on third-party library maintenance
   - **Mitigation**: Library is npm-published, can fork if needed

4. **Hardcoded Credentials**: Development credentials visible in code
   - **Issue**: Security concern in init() function
   - **Recommendation**: Move to secure configuration
