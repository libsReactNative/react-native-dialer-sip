## Component Specifications

### 1. Endpoint Lifecycle

#### 1.1 Initialization

**Function**: `init()`

**Type**: Thunk action (async)

**Responsibilities**:
1. Create Endpoint instance
2. Start endpoint with configuration
3. Subscribe to events
4. Initialize push notifications (iOS)
5. Dispatch INIT action

**Configuration**:
```javascript
{
  "name": "MyUserName",
  "username": "50363",
  "password": "pass50363",
  "domain": "172.16.104.17",
  "regServer": "",
  "proxy": null,
  "transport": "UDP",
  "regTimeout": 3600,
  "regHeaders": {},
  "regOnAdd": true,
  "service": {
    "ua": "siptest"
  },
  "network": {
    "useAnyway": true,
    "useWifi": true,
    "use3g": true,
    "useEdge": true,
    "useGprs": true,
    "useInRoaming": true,
    "useOtherNetworks": true
  }
}
```

**Event Subscriptions**:
```javascript
endpoint.on("registration_changed", (account) => {
  dispatch(onAccountChanged(account))
})

endpoint.on("connectivity_changed", (available) => {
  dispatch(onConnectivityChanged(available))
})

  dispatch(onCallReceived(call))

  dispatch(onCallChanged(call))

  dispatch(onCallScreenLocked(call))
```

**iOS Specific**:
- Initialize push notifications
- Setup CallKit (commented out)
- AppState listener for background registration

**State Transformation**:

#### 1.2 Destruction

**Function**: `destroy()`

**Type**: Thunk action (async)

**Responsibilities**:
1. Get endpoint and accounts from state
2. Delete all accounts
3. Remove all event listeners

**Implementation**:
```javascript
export function destroy() {
  return async function (dispatch, getState) {
    const {endpoint, accounts} = getState().pjsip

    for (const id in accounts) {
      if (accounts.hasOwnProperty(id)) {
        endpoint.deleteAccount(accounts[id])
      }
    }

    endpoint.removeAllListeners()
  }
}
```

### 3. Call Operations

#### 3.1 Make Call

**Function**: `makeCall(destination, account = null)`

**Type**: Thunk action (async)

**Inputs**:
- `destination`: string (phone number or SIP URI)
- `account`: Account (optional, uses first available)

**Process**:
```javascript
1. Get accounts from state
2. If no account provided:
   - Find first account in accounts map
   - If no accounts: navigate to call screen with error
3. If iOS: endpoint.deactivateAudioSession()
4. call = endpoint.makeCall(account, destination)
5. Navigate to call screen with call promise
```

### 4. Push Notification Integration (iOS)

#### 4.1 Initialize Push Notifications

**Function**: `initPushNotifications()`

**Type**: Thunk action (async)

**Responsibilities**:

**Listeners**:
```javascript

VoipPushNotification.addEventListener('notification', (notification) => {
  const {endpoint, accounts} = getState().pjsip
  for (const id in accounts) {
    if (accounts.hasOwnProperty(id)) {
      endpoint.registerAccount(accounts[id], true)
    }
  }
})

// VoIP token registration

  resolve(token)
```

**Token Storage**:

#### 4.2 Background Registration

**Trigger**: AppState change to 'background' or 'active'

**Process**:
```javascript
AppState.addEventListener('change', (nextAppState) => {
  if (nextAppState == 'background' || nextAppState == 'active') {
    const accounts = getState().pjsip.accounts
    for (const id in accounts) {
      if (accounts.hasOwnProperty(id)) {
        endpoint.registerAccount(accounts[id], nextAppState == 'active')
      }
    }
    dispatch({type: CHANGED_APP_STATE, payload: {appState: nextAppState}})
  }
})
```

## State Machine

### Account States

```
[Created] → [Registering] → [Registered]
                ↓               ↓
          [Registration Failed] [Re-registering]
```

### Call States

```
[Null] → [Initiated] → [Incoming] → [Active] → [Disconnected]
                          ↓            ↓
                      [Declined]   [Held]
                                      ↓
                                  [Terminated]
```

## Data Flow Diagrams

### Outgoing Call Flow

```
User → DialerViewport → makeCall() → pjsip.makeCall()
                                        ↓
                                    Endpoint.makeCall()
                                        ↓
                                    call_received event
                                        ↓
                                    onCallReceived()
                                        ↓
                                    Navigation.goTo(call screen)
                                        ↓
                                    CallScreen renders
```

### Incoming Call Flow (Foreground)

```
SIP Server → Endpoint → call_received event
                              ↓
                        onCallReceived()
                              ↓
                        Navigation.goTo(call screen)
                              ↓
                        CallScreen renders
                              ↓
                        User: answerCall()
```

### Incoming Call Flow (Background, iOS)

```
SIP Server → APNs → VoIP Push Notification
                        ↓
                  notification event
                        ↓
                  Re-register accounts
                        ↓
                  App wakes up
                        ↓
                  Endpoint → call_received
                        ↓
                  onCallReceived()
                        ↓
                  Navigation + UI
```

## Performance Considerations

### State Lookup

- Accounts: O(1) via map `{[id]: Account}`
- Calls: O(1) via map `{[id]: Call}`

### Event Handling

- All endpoint events dispatch to Redux immediately
- UI updates via React-Redux connect()

### Memory Management

- Endpoint destroyed on app destroy
- All listeners removed
- Calls removed from state on termination

## Security Considerations

### ⚠️ Known Issues

1. **Hardcoded Credentials**:
   ```javascript
   // In init() - DEVELOPMENT ONLY
   username: "50363",
   password: "pass50363",
   domain: "172.16.104.17"
   ```
   **Recommendation**: Move to secure configuration/environment variables

2. **Push Tokens**: Stored in Redux (in-memory only, not persisted)

3. **No Encryption**: SIP credentials sent in plain configuration
