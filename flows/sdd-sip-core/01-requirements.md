## Overview

The SIP Core module provides Voice over IP (VoIP) communication capabilities using the PJSIP protocol via the react-native-sip2 library. This module handles SIP endpoint management, account lifecycle, call operations, and platform-specific integrations (iOS VoIP push notifications, CallKit).

## Stakeholders

- **Primary Users**: Application developers integrating SIP calling
- **End Users**: Mobile app users making/receiving VoIP calls
- **Dependencies**: react-native-sip2, VoipPushNotification (iOS), PushNotificationIOS

## Business Requirements

### BR-1: SIP Account Management

Users must be able to configure and manage SIP accounts for making and receiving calls.

**Acceptance Criteria:**
- Create SIP account with credentials (username, password, domain)
- Delete existing SIP account
- Account automatically registers with SIP server on creation
- Account re-registers periodically (timeout: 3600 seconds)

### BR-2: Call Operations

Users must be able to make and receive calls with full control.

**Acceptance Criteria:**
- Initiate outgoing call to destination number
- Answer incoming call
- Reject/decline incoming call
- Hangup active call
- Hold/unhold call
- Mute/unmute microphone
- Switch between speaker and earpiece
- Transfer call (blind and attended)
- Redirect call to another destination
- Send DTMF tones during call

### BR-3: Multiple Account Support

Users may configure multiple SIP accounts simultaneously.

**Acceptance Criteria:**
- Support multiple SIP accounts
- Default account selection for outgoing calls
- Display all registered accounts

### BR-4: Incoming Call Notifications

Users must receive incoming call notifications even when app is in background.

**Acceptance Criteria:**
- iOS: Receive VoIP push notifications
- Display incoming call UI
- Re-register accounts when receiving push in background

### BR-5: Network Resilience

Application must handle network connectivity changes gracefully.

**Acceptance Criteria:**
- Detect connectivity changes
- Support WiFi, 3G, Edge, GPRS networks
- Configurable network usage policies
- Handle roaming scenarios

## Functional Requirements

### FR-1: Endpoint Initialization

**Description**: Initialize SIP endpoint with configuration.

**Inputs:**
- Configuration object (credentials, network settings, service settings)

**Process:**
1. Create new Endpoint instance
2. Start endpoint with configuration
3. Subscribe to endpoint events
4. Store endpoint state in Redux

**Outputs:**
- Initialized endpoint
- Initial state (accounts, calls, connectivity)

**Events Subscribed:**
- `registration_changed` → Account registration state
- `connectivity_changed` → Network connectivity
- `call_received` → Incoming call
- `call_changed` → Call state update
- `call_terminated` → Call ended
- `call_screen_locked` → Screen lock state

### FR-2: Account Creation

**Description**: Create new SIP account with configuration.

**Inputs:**
- Configuration: username, password, domain, transport, proxy, regTimeout, regHeaders

**Process:**
1. Get endpoint instance from state
2. Get push tokens (iOS only)
3. Build contact URI params with tokens
4. Call `endpoint.createAccount()` with config
5. Dispatch ACCOUNT_CREATED action

**Outputs:**
- Account object

**Contact URI Params:**
- iOS: `;app-id=com.carusto.mobile.app;pn-voip-tok={token};pn-im-tok={token}`
- Android: `;im-type=sip`

### FR-3: Account Deletion

**Description**: Remove SIP account from endpoint.

**Inputs:**
- Account object

**Process:**
1. Get endpoint instance
2. Call `endpoint.deleteAccount(account)`
3. Dispatch ACCOUNT_DELETED action
4. Navigate to settings screen

**Outputs:**
- Account removed from state

### FR-4: Make Call

**Description**: Initiate outgoing call.

**Inputs:**
- Destination number (string)
- Account (optional, uses default if not provided)

**Process:**
1. Get accounts from state
2. Select default account if none provided
3. Deactivate audio session (iOS)
4. Call `endpoint.makeCall(account, destination)`
5. Navigate to call screen

**Outputs:**
- Call object
- Navigation to call screen

### FR-5: Answer Call

**Description**: Answer incoming call.

**Inputs:**
- Call object

**Process:**
1. Get endpoint instance
2. Call `endpoint.answerCall(call)`

**Outputs:**
- Call answered

### FR-6: Hangup Call

**Description**: Terminate active call.

**Inputs:**
- Call object

**Process:**
1. Get endpoint instance
2. Call `endpoint.hangupCall(call)`

**Outputs:**
- Call terminated

### FR-7: Call Hold/Unhold

**Description**: Place call on hold or resume.

**Inputs:**
- Call object

**Process:**
- Hold: `endpoint.holdCall(call)`
- Unhold: `endpoint.unholdCall(call)`

### FR-8: Call Mute/Unmute

**Description**: Mute or unmute microphone.

**Inputs:**
- Call object

**Process:**
- Mute: `endpoint.muteCall(call)`
- Unmute: `endpoint.unmuteCall(call)`

### FR-9: Audio Route Selection

**Description**: Switch audio between speaker and earpiece.

**Inputs:**
- Call object

**Process:**
- Speaker: `endpoint.useSpeaker(call)`
- Earpiece: `endpoint.useEarpiece(call)`

### FR-10: Call Transfer

**Description**: Transfer call to another destination or call.

**Inputs:**
- Call object
- Destination (blind transfer) or DestinationCall (attended transfer)

**Process:**
- Blind: `endpoint.xferCall(account, call, destination)`
- Attended: `endpoint.xferReplacesCall(call, destinationCall)`

### FR-11: Call Redirect

**Description**: Redirect incoming call to another destination.

**Inputs:**
- Call object
- Destination number

**Process:**
1. Get accounts
2. Call `endpoint.redirectCall(account, call, destination)`

### FR-12: DTMF Tones

**Description**: Send DTMF tones during call.

**Inputs:**
- Call object
- Key (string)

**Process:**
- Call `endpoint.dtmfCall(call, key)`

### FR-13: Push Notification Integration (iOS)

**Description**: Handle VoIP push notifications for incoming calls.

**Inputs:**
- Push notification

**Process:**
1. Request VoIP and IM permissions
2. Obtain tokens
3. Store tokens in state
4. On notification: re-register accounts
5. Dispatch INIT_PN action with tokens

### FR-14: Endpoint Destruction

**Description**: Clean up endpoint and remove all accounts.

**Inputs:**
- None

**Process:**
1. Get endpoint and accounts from state
2. Delete all accounts
3. Remove all event listeners

### FR-15: Network Configuration Change

**Description**: Update network settings.

**Inputs:**
- Configuration: foreground, network flags

**Process:**
1. Get endpoint
2. Call `endpoint.changeNetworkConfiguration()`
3. Call `endpoint.changeServiceConfiguration()`
4. Dispatch CHANGED_SETTINGS
5. Navigate to settings screen

## Data Requirements

### DR-1: Redux State Structure

```javascript
{
  pjsip: {
    endpoint: Endpoint,           // SIP endpoint instance
    endpointSettings: Object,     // Service/network settings
    connectivity: boolean,        // Network available
    isScreenLocked: boolean,      // Screen lock state
    appState: 'active'|'background',
    tokens: {
      voip: string|null,          // VoIP push token
      im: string|null             // IM push token
    },
    accounts: {                   // Map of account ID → Account
      [id]: Account
    },
    calls: {                      // Map of call ID → Call
      [id]: Call
    }
  }
}
```

### DR-2: Account State

Accounts stored as map keyed by account ID for O(1) lookup.

### DR-3: Call State

Calls stored as map keyed by call ID for O(1) lookup.

## Constraints

### C-1: State Persistence

Application state is **not persistent** when user closes app. Endpoint must be re-initialized on app start.

### C-2: CallKit Limitation

CallKit does not support multiple simultaneous calls. Architecture accepts single active call constraint.
