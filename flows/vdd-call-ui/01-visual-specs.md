## Overview

The Call UI provides the visual interface for managing SIP calls, including call state display, user actions, and multi-call handling. The design emphasizes smooth animations and clear visual feedback during call lifecycle.

## Visual Design Principles

1. **State-Driven Layout**: UI elements animate to different positions based on call state
2. **Minimal Cognitive Load**: Clear visual hierarchy, essential information prominent
3. **Smooth Transitions**: Animated changes prevent jarring state switches
4. **Touch-Friendly**: Large touch targets for critical actions (answer, hangup)
5. **Color Semantics**: Green (answer/active), Red (hangup), Yellow (redirect/hold)

## Animation Specifications

### State Transitions

**Incoming → Active (Answer)**
```
Avatar: opacity 1 → 0, offset moves down
Actions: opacity 0 → 1, offset moves up
Controls: hangup button slides to answer position
Duration: 300ms
Easing: ease-in-out
```

**Active → Terminated (Hangup)**
```
Avatar: opacity 0 (stays faded)
Actions: opacity 1 → 0, offset moves down
State: remains visible
Controls: hangup button stays
Duration: 300ms
Easing: ease-in-out
```

### Animated Values

```javascript
// Animated in CallScreen
infoOffset: Animated.Value    // Vertical position of call info
avatarOpacity: Animated.Value // Avatar fade (0-1)
avatarOffset: Animated.Value  // Avatar vertical position
stateOffset: Animated.Value   // State text vertical position
actionsOpacity: Animated.Value // Actions fade (0-1)
actionsOffset: Animated.Value  // Actions vertical position
buttonsOffset: Animated.Value  // Controls vertical position
```

### Layout Calculation

**Incoming Call Layout:**
```javascript
infoOffset = switchHeight + switchOffset + space
avatarOffset = infoOffset + infoHeight + space
stateOffset = avatarOffset + avatarHeight + space
buttonsOffset = max(stateOffset + stateHeight + space, screenHeight - buttonsHeight - 56)
actionsOffset = stateOffset + stateHeight + space

// Avatar visible, actions faded
avatarOpacity = 1
actionsOpacity = 0
```

**Active Call Layout:**
```javascript
infoOffset = switchHeight + space
stateOffset = infoOffset + infoHeight + space
actionsOffset = stateOffset + stateHeight + space
buttonsOffset = actionsOffset + actionsHeight + space
avatarOffset = infoOffset + infoHeight + space

// Avatar faded, actions visible
avatarOpacity = 0
actionsOpacity = 1
```

## Interaction Patterns

### Making a Call

1. User enters number in DialerViewport
2. Taps "Call" button
3. CallScreen opens with "Please wait" message
4. Call initializes (Promise resolves)
5. UI transitions to "Calling..." state
6. Call connected → transitions to active state

### Answering a Call

1. Incoming call → CallScreen opens
2. UI shows incoming state (avatar visible, answer button)
3. User taps Answer
4. UI transitions to active state
5. Call conversation begins

### During Active Call

**Primary Actions (always visible):**
- Hangup (red button)

**Secondary Actions (ViewPager Page 1):**
- Tap Add → Opens dialer modal to add another call
- Tap Mute → Mutes microphone, icon changes
- Tap Speaker → Switches to speaker, icon changes
- Tap Transfer → Opens transfer modal
- Tap Hold → Puts call on hold, icon changes
- Tap DTMF → Opens DTMF keypad modal

**Navigation:**
- Swipe ViewPager → Page 2 (additional actions)
- Tap page indicator → Switch pages

### Ending a Call

1. User taps Hangup
2. Call terminates
3. UI transitions to terminated state
4. After 3 seconds: navigates away
   - If other calls: navigate to active call
   - If no calls: navigate back to previous screen

### Multi-Call Handling

**Scenario: Second incoming call during active call**
1. User on active call
2. Second call arrives
3. IncomingCallModal appears over call screen
4. User can:
   - Answer → Current call on hold, new call active
   - Decline → Reject new call, continue current

**Scenario: Multiple active calls**
1. User has multiple calls
2. CallParallelInfo strips show other calls
3. Tap strip → Switch to that call
4. Current call info updates
